Vocabulary recap

  • Modal frame: red header (top-right corner rounded as locked) + dark body + 3-sided red border (left, right, bottom) — visually a U-frame that closes at the header. Differentiates a modal from persistent panels (which use left-bar only).
  • Backdrop: radial dim from centre to edges; the scene behind continues to render but is darkened. Modal blocks input.
  • Action buttons (LCARS end-cap pattern): hard-cornered buttons sit between two rounded end-caps that bookend the row. Caps appear only at the row's outer edges — never between buttons. Maps to STButton.SetUseEndCaps(left, right) per-button: leftmost gets (1,0), rightmost gets (0,1), anything in between gets (0,0). Small gap (6px) between cap and the first/last button gives breathing room. Caps inherit the panel's primary colour (red); button body reflects state (filled g_kSTMenu2NormalBase purple primary, transparent with horizontal outlines secondary, grey disabled).
  • Single Antonio weight (400) throughout — header, body, buttons. State is colour-only (no 600 weight for emphasis, no italic except the rare disabled-row case).
  • Tabs (when present): reuse the menu-row vocabulary with ▸ chosen caret. Tabs are navigation, buttons are commit-actions — kept visually distinct via the pill shape.

SDK alignment

  • Maps to ModalDialogWindow.Run(title, _, btn1Text, btn1Event, btn2Text, btn2Event) at App.py:7538. Title comes from arg 1; up to two buttons baked in by the call.
  • Inner stylized window accessible via pModalDialogWindow.GetFirstChild() as an STStylizedWindow — content code adds children to it (paragraphs, sub-panes, lists) using AddChild(child, x, y, layer).
  • The simple-modal example mirrors the only SDK use site (KeyboardConfig.py:545): title + body paragraph + 1–2 action buttons.
  • The complex-modal example demonstrates that the same chrome scales to the Quick Battle Setup (mentioned in bridge-mode.md §115). The body's three-column structure would be assembled by Python via positioned child panes — chrome unchanged.

RmlUi compatibility

  • border-left / border-right / border-bottom individually — all supported.
  • End-cap rounding: border-radius: 12px 0 0 12px / 0 12px 12px 0 on the cap children — per-corner shorthand, supported.
  • Backdrop dim — implemented at runtime as a full-screen linear-gradient decorator at low alpha behind the modal element. The radial dimming shown in the mockup is for visual demonstration; RmlUi supports radial-gradient too if we want the focal centre.
  • No CSS variables, no grid — only flexbox and explicit colours, matching the rest of the locked vocabulary.
