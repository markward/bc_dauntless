What's in / what's deferred

  • Phase 1 (now): locked panel chrome + label + hull integrity bar. Label shows the target's name when a target is selected, the literal string PLAYER for the player panel, or NO TARGET when nothing is selected.
  • Deferred (Phase 2): ship sprite from game/data/icons/Ships/<HullClass>.tga, six-face shield visualization (ring or cross treatment), damage-icon overlays from the engineering glyph set. Rendering scope sits with the C++ engine.
  • SDK plumbing stays unchanged: App.ShipDisplay_Create() still wraps both panels; SetShipDisplay() / SetEnemyShipDisplay() still register them with the tactical window. The data feed and event handling are wired; only the painted visual is reduced.

Hull bar colour states (kept)

  • Healthy rgb(50, 210, 80) — green fill, full width
  • Damaged rgb(255, 200, 60) — yellow fill below 75%
  • Critical rgb(255, 80, 40) — red fill below 25%
  • Empty track rgb(40, 40, 40) — base bar background
  • No target — track at reduced opacity, no fill
