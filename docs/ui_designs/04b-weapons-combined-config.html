Click the hamburger and the status view is replaced by the weapon-settings view — Torpedoes (Type/Spread), Phasers (Intensity), and the Tractor/Cloak toggles. Behaviour & colour mapping

  • Hamburger toggle (top-left of the body) — by default the panel shows the status view exactly as today (60%-scaled ship silhouette + per-mount phaser-arc icons + speed label); the only addition is the hamburger. Clicking it hides the status view and reveals the weapon-settings view below; clicking again flips back. Tooltip on hover ("Weapon settings" / "Hide settings").
  • Torpedoes row = two buttons. Button 1 (torp-btn--type) shows the selected type + remaining quantity, e.g. Photon (250) — click cycles carried types (Photon, Quantum…). On a single-type hull it renders torp-btn--static (grey, not clickable). Button 2 (torp-btn--spread) shows just the spread word — Single / Dual / Quad — click cycles the options the loadout can fire; drives the volley count + fan-out pattern.
  • Intensity (row-cycle, under the Phasers heading) — cycles Full / Light on every hull (PhaserSystem.SetPowerLevel PP_HIGH / PP_LOW).
  • Tractor & Cloak (sys-row, salmon Menu1) — two On/Off buttons sharing one row, ~50/50. Each button renders only when its subsystem is equipped; if the hull has just one (e.g. Galaxy has a tractor but no cloak), the remaining button fills the row at 100% width via flex:1.
  • Toggle state: no coloured value text — an On device fills its button with the salmon border colour (sys-btn--on); Off shows the translucent salmon base. Value text stays neutral in both states.
  • Section gating (whole sections, not just rows):
    • The Torpedoes section (heading + type/spread row) renders only if the ship has at least one torpedo launcher.
    • The Phasers section (heading + Intensity row) renders only if the ship has phasers.
    • Each of Tractor / Cloak renders only if that subsystem is equipped (already shown).
    • If a ship has nothing to configure at all, the hamburger toggle isn't shown.

Layer rule (from 04): rows inside a labelled section drop to Menu3 pink; standalone toggles stay at the panel's Menu1 salmon — colour alone signals “section sub-control” vs “top-level toggle”.
