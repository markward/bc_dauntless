Colour mapping (all SDK globals)

  • Panel chrome (header, border): g_kSTMenu1NormalBase rgb(216,94,86) — Menu1, because Weapons is a top-level HUD panel.
  • Section subheaders ("Torpedoes", "Phasers", torpedo count "250"): g_kTitleColor rgb(255,154,2) — yellow-gold.
  • Cycle rows (Type / Spread / Intensity) — sit inside a section. Use Menu3 pink:
    • Background: g_kSTMenu3HighlightedBase rgb(246,147,204) at 35% α
    • Left accent: g_kSTMenu3NormalBase rgb(207,96,159)
  • Toggle rows (Tractor / Cloak) — standalone, not inside a section. Use Menu1 salmon (same colour family as the panel chrome — distinguishes them visually from in-section controls):
    • Background: g_kSTMenu1NormalBase rgb(216,94,86) at 45% α
    • Left accent: g_kSTMenu1NormalBase rgb(216,94,86) solid
  • All row text: g_kTitleColor rgb(255,154,2). Value rendered in italic to differentiate from the label while staying in one colour.
  • "Off" state value (Cloak Off): g_kSubsystemDisabledColor rgb(153,153,153).

Layer rule (panel-relative)

Rows pick their colour layer based on their position within the panel:

  • Rows inside a labelled section (Torpedoes / Phasers) drop one layer down → Menu3 pink.
  • Rows standalone (not inside any subsection) stay at the panel's level → Menu1 salmon.

This gives a visual hierarchy without needing extra chrome — the row colour itself reads as "this is a section sub-control" vs "this is a top-level control".

Row behaviour (corrected — none are dropdowns)

  • Type — click cycles through torpedo types carried on the hull (Photon, Quantum, etc. — list comes from the hardpoint).
  • Spread — click cycles through spread options offered by the loadout (Single, 2, 3, …).
  • Intensity — click cycles between hardcoded Full / Low (same on every ship).
  • Tractor — click toggles On / Off. Draws from Main Battery while On.
  • Cloak — click toggles On / Off (row only renders on hulls with cloak equipped). Draws from Reserve Battery while On.
  • No expand chevrons. Clicking the row cycles or toggles in place; no popout menu.

Three row behaviours

  • Cycle rows (Type, Spread, Intensity): clicking cycles the value to the next option in g_lTorpedoTypes / g_lTorpedoSpreads / g_lPhaserIntensities. Bullet is a purple-outlined cell with a small gold chevron indicating "more values available".
  • Toggle on (Tractor): filled gold bullet with soft glow. Value "On" in gold. Draws continuously from Main Battery.
  • Toggle off (Cloak): hollow purple-outline bullet. Value "Off" in dim grey. Row only renders if the hull has a cloak system equipped (per tactical-spec.md).

SDK alignment

  • App.WeaponsDisplay_Create(w, h) at App.py:8468. Inherits STStylizedWindow, panel chrome applies automatically.
  • Created by CreateWeaponsDisplay(pTacticalWindow) at TacticalMenuHandlers.py:775, registered via pTacticalWindow.SetWeaponsDisplay(...).
  • Companion widget CreateWeaponsControl at line 830 handles the click-to-cycle event wiring.
  • Cycle values come from the loaded campaign's torpedo/phaser definitions — the panel reads them, doesn't define them.
  • Cloak row visibility is determined per-hardpoint at ship-spawn time.

Speed / Phaser Arc — placeholder (descoped)

Like Shields, the Speed / Phaser-Arc panel needs a custom renderer: per-tube torpedo readiness indicators (visible in the original game's screenshot 1 as a row of glowing tube blocks) plus a horizontal phaser-arc readout showing where the phasers will hit. Phase 1 keeps only the panel chrome + a centred numeric speed label. Phase 2 brings in the tube/arc renderer.
